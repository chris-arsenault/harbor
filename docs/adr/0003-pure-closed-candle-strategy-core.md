# 0003 - Pure Closed-Candle Strategy Core

- Status: Accepted
- Date: 2026-06-16

## Context

The strategy can silently become invalid if live execution and backtests observe different data or if the implementation reads the currently forming candle. The source specification names repainting as the primary failure mode.

## Decision

Harbor keeps the strategy core as a pure, deterministic module that consumes closed candles, session levels, current state, and config, then emits decisions. Live trading, backtesting, optimization, and paper variants all call the same core.

## Alternatives considered

- **Strategy logic inside the live feed service** - shortens the first implementation, but makes backtesting and optimizer results diverge from live execution.
- **Separate live and backtest strategies** - allows environment-specific shortcuts, but creates duplicated semantics and lookahead risk.
- **Indicator-style mutable callbacks** - can be convenient for streaming logic, but makes deterministic replay and fixture testing harder.

## Consequences

Feed code must emit only closed candles. Tests must cover sweep, FVG, stop, target, sizing, window, and kill-switch behavior through deterministic fixtures. UI overlays are generated by backend facts rather than frontend recomputation.
